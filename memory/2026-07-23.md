# 2026-07-23 笔记

## 凌晨 railway_file_server.py 部署救援

### 问题背景
爹部署到 Railway 的 `railway_file_server.py` 报 502 错误，打不开页面。线上版本在 `d758674`（带了改密码功能+profileModal）。

### 关键发现
- **Railway 部署大小的限制**：文件包不能超过一定大小。之前的版本 **a08194d**（85KB 无改密码+profileModal）能正常部署。
- **CRLF vs LF**：git 存储用 LF（85KB），Windows 工作区用 CRLF（90KB），差了~5KB。实际部署到 Railway 用的是 LF 版本，所以本地看到的 90KB 不代表部署时也这么大。
- **授权游戏功能**（`da65b19` 开始）加上改密码/profileModal 导致 _ADMIN 模板膨胀，超过了 Railway 的部署上限。

### 关键操作
1. 认识到我们陷入"改了什么"的死胡同，爹要求看中午日志
2. 查出 `a08194d` commit（85KB）是爹中午确认能用的版本
3. 推回 `a08194d` → 线上恢复 ✅（爹反馈部署成功，能用）
4. Railway 自动从 main 分支 GitHub 拉取部署

### 冲突点
- 爹想要：改密码功能 + 授权游戏
- 现实限制：每个版本 _ADMIN 模板占了绝大部分体积（~75KB），加任何新功能都可能超限
- 方案：以后的 feature 要么不加到 _ADMIN 模板里，要么外部引用 JS/CSS
