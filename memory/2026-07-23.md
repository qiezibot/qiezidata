### 关键发现
- **Railway 部署大小的限制**：文件包不能超过一定大小。之前的版本 **a08194d**（85KB 无改密码+profileModal）能正常部署。
- **CRLF vs LF**：git 存储用 LF（85KB），Windows 工作区用 CRLF（90KB），差了~5KB。实际部署到 Railway 用的是 LF 版本，所以本地看到的 90KB 不代表部署时也这么大。
- **授权游戏功能**（`da65b19` 开始）加上改密码/profileModal 导致 _ADMIN 模板膨胀，超过了 Railway 的部署上限。

3. 推回 `a08194d` → 线上恢复 ✅（爹反馈部署成功，能用）
4. Railway 自动从 main 分支 GitHub 拉取部署

### 冲突点
- 爹想要：改密码功能 + 授权游戏

### 关键教训
- Rust/Python/AI coding agent 都可能写出类似 "backslash before template" 的错误，尤其是多人多次编辑后
- CRLF vs LF 在 git diff 中不显示差异，但 Python 的 byte-level replace 会匹配失败
- `edit` 工具在大型 Python 三引号模板中的字符串替换容易翻车（模版字符串里可能包含引号、反斜杠等特殊字符），二进制替换更可靠
- 删除用户用的是事件委托（`userTableBody.onclick`），不是 inline onclick，所以不会因为 HTML 模板重新渲染而丢失
- 所有的按钮删除、设为管理员、取消管理员都走事件委托 + `button.closest()` 判断类名

### 有用命令
- 二进制替换：`c = open('file.py','rb').read(); c = c.replace(b'old_bytes', b'new_bytes', 1); open('file.py','wb').write(c)`
- 检查字节：`print(c[idx-20:idx+80].hex(' '))` 检查 CRLF (`0d 0a`) vs LF (`0a`)
- 检查模板中 `loadUsers` 字符串是否被 server 正确提供：用浏览器 `fetch('/api/admin/users').then(r=>r.text()).then(t=>t.match(/del-btn|取消管理员/g))`
