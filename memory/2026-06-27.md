## 22:01~22:07 二值化工具启动问题（pythonw.exe ADB失效）

### 问题
- 用户打开二值化工具时有两个窗口（GUI + 黑色cmd后台）
- 改用 `pythonw.exe`（无控制台Python）后，ADB设备检测失败，显示"无设备"
- 桌面快捷方式已改为 pythonw.exe 路径
- `adb devices` 命令行下显示设备在线 `1aabaaeb	device`
- 但工具内 `subprocess.run([ADB_EXE,"devices"])` 返回无设备

### 根因
- `pythonw.exe` 是Windows子系统GUI程序，不继承用户环境变量和PATH
- ADB路径虽然写死了绝对路径，但 `pythonw.exe` 启动的进程可能缺少某些环境变量导致subprocess失败
- **不用pythonw.exe，用 python.exe + start /B /MIN 最小化启动**

### 解决
- 桌面快捷方式改为：cmd.exe /c start /B python.exe script
- 启动.bat 也改为 start /B /MIN python.exe
- 启动后cmd窗口一闪而过，不像pythonw.exe那样完全无进程问题

## 23:07~23:21 QQ机器人恢复全过程

### 问题症状
- QQ机器人显示"灵魂不在线"，收不到QQ消息
- 通过控制面板发送消息也不行

### 排查发现
1. **插件路径有多个版本**：
   - `@sliverp/qqbot`（`node_modules/`下）——dist目录丢失，需手动 `npx tsc` 编译
   - `openclaw-qqbot`（`extensions/openclaw-qqbot/` 下，来自 `@tencent-connect/openclaw-qqbot` v1.7.1）——这个是真正工作的版本

2. **配置文件反复被冲掉**：
   - `config.patch` 在 `openclaw.json` 无 `plugins`/`channels` 顶层字段时不识别
   - `config.apply` 会替换整个配置文件，丢掉 gateway.auth、models 等关键配置
   - `plugins.entries.qbot` 的schema要求 `additionalProperties = false`，不接受config字段里任意名称的字段（因为openclaw.plugin.json没声明configSchema）
   - SIGUSR1热重启时gateway自己会重建openclaw.json，可能丢掉写入的内容

3. **最终根因：字段名大小写错误** ⭐
   - 插件 `config.js:135` 读 `cfg.channels?.qqbot`
   - `listQQBotAccountIds()`：`const qqbot = cfg.channels?.qqbot; if (qqbot?.appId) { ids.add() }` — 检查的是 `appId`（小写d）
   - `resolveQQBotAccount()` 第184行：`appId = normalizeAppId(qqbot?.appId)` — 也是小写d
   - 但之前配置文件一直写的 `"appID": "1904006743"`（**大写D**）
   - 所以插件启动时 `qqbot?.appId` 一直返回undefined → `if (!account.appId || !account.clientSecret)` 抛错 → WebSocket不连

### 解决步骤
1. `node -e` 直接写 `openclaw.json`（绕过 config.patch 的限制）
2. 字段名写 `appId` 而不是 `appID`
3. 文件级写入后用 `gateway.restart`（SIGUSR1）重启
4. 顺带编译了 `@sliverp/qqbot` 的 dist/ 目录
